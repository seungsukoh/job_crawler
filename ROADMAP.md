# 실행 로드맵

## 운영 원칙

이 프로젝트는 무료 운영으로 시작한다. 초기 목표는 완성도 높은 채용 플랫폼을 만드는 것이 아니라, 대학생이 직무와 마감일 기준으로 공고를 찾고 저장할 이유가 있는지 검증하는 것이다.

핵심 원칙:

- 공고 수보다 데이터 신뢰도와 최신성을 우선한다.
- 안전하게 수집 가능한 소스만 사용한다.
- 대형 민간 채용 플랫폼 무단 크롤링은 MVP 범위에서 제외한다.
- 사용자 검색 요청에서는 외부 사이트를 직접 크롤링하지 않는다.
- 크롤링은 스케줄러 또는 관리자 수동 작업으로만 실행하고, 검색은 내부 DB를 조회한다.
- 처음에는 Cloudflare Pages, Render Free, Supabase Free, GitHub Actions 조합으로 간다.
- 기능은 작게 만들고, 매주 지표와 리스크를 보고 조정한다.

## 6주 MVP 계획

| 주차 | 목표 | 산출물 | 완료 기준 |
|---|---|---|---|
| 1주차 | 프로젝트 기반 구축 | Next.js, FastAPI, DB 연결, 샘플 데이터 | 로컬에서 web/api/db 실행, 샘플 공고 조회 가능 |
| 2주차 | 공고 탐색 MVP | 목록, 상세, 검색, 필터, 원문 링크 | 샘플 데이터로 검색/필터/상세/원문 이동 가능 |
| 3주차 | 안전 수집 파이프라인 1차 | Source Registry, API/RSS fetcher, 정규화, 중복 제거 | 저위험 소스 1~2개 수집 성공 |
| 4주차 | 저장 및 마감 관리 | 관심 공고 저장, 저장 목록, 마감 임박 표시 | 사용자가 공고 저장 후 마감순 관리 가능 |
| 5주차 | 무료 배포 | Cloudflare Pages, Render Free, Supabase Free, GitHub Actions | 외부 URL 접속, 수동/예약 수집 가능 |
| 6주차 | 베타 검증 | 데이터 품질 확인, 모바일 UX 정리, 피드백 반영 | 대학생 10~20명 테스트와 지표 수집 |

## 우선순위

### P0

- 공고 목록/상세 조회
- 직무 키워드 검색
- 마감일 기준 필터
- 신입/인턴/채용연계형/졸업예정자 가능 태그
- 원문 링크 이동
- 관심 공고 저장
- 마감 임박 표시
- 중복 공고 병합
- 수집 시각/원문 확인 시각 표시
- 공공 API/RSS/허용된 공개 페이지 기반 수집
- GitHub Actions 또는 관리자 명령 기반 주기 수집

### P1

- 지역/고용형태/직무 카테고리 facet
- 저장 공고 상태: `검토 중`, `지원 예정`, `지원 완료`
- 관리자용 수집 현황 화면
- 크롤링 실패 로그
- 마감일 누락 공고 검수 큐
- GitHub Actions 예약 수집
- 마감 임박 공고 재검증 작업

### P2

- 관심 키워드 알림
- 이메일 알림
- 학교별/전공별 추천 필터
- 관리자 수동 등록 기능
- 소스별 신뢰도 표시

## 개발 관리 기준

브랜치:

- `main`: 배포 가능한 안정 브랜치
- 작업 브랜치: `feat/issue-12-job-list-api`, `fix/issue-18-crawler-timeout` 형식

이슈 타입:

- `Epic`
- `Feature`
- `Task`
- `Bug`
- `Security`
- `Ops`

PR 기준:

- 하나의 PR은 하나의 이슈를 닫는 단위로 유지한다.
- DB schema, 크롤링 정책, 인증/보안 변경은 필수 리뷰 대상으로 본다.
- 테스트 실패 PR은 merge하지 않는다.

Definition of Done:

- 기능이 이슈 완료 조건을 만족한다.
- API 또는 UI가 정상 동작한다.
- 타입 체크, 린트, 테스트가 통과한다.
- DB 변경이 있으면 migration이 포함된다.
- 환경 변수가 추가되면 `.env.example`과 문서가 갱신된다.
- 크롤링 관련 기능은 robots.txt, rate limit, timeout 정책을 준수한다.
- 로그에 민감 정보가 남지 않는다.
